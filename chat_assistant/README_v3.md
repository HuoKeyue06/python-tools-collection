# Agent CLI v3.0 --具备工具调用能力的智能助手
## 版本概述：
v3.0在v2.0模块化架构的基础上，实现了 “Function Calling(函数调用)” 机制。Agent现在能够：
- 自动识别用户意图，决定是否需要调用工具
- 识别工具调用的参数，并调用工具
- 返回工具调用结果给用户
**核心突破**；模型从“只会说话的聊天机器人”升级为“具备工具调用能力的智能助手”
## 新增功能：
### 1. Function Calling(函数调用)闭环
- 改造'ZhipuClient',新增'chat_with_tools'方法，支持携带'tools'参数
- 在'ai-cli.py'中实现完整的工具调用闭环：
1. 发送用户问题 + 工具列表
2. 解析模型返回的'tool_call'
3. 调用'ToolRegistry.execute()'执行工具调用
4. 将工具结果以'role:"tool"'格式回传
5. 再次调用模型生成最终自然语言回复
### 2.智能工具选择
- 用户问“现在几点”--- 模型自动调用"get_current_time" 
- 用户问“计算123*123” --- 模型自动调用"calculator"
- 用户问普通问题 --- 模型自动回答，不需要调用工具

## Function Calling 流程图
用户: "现在几点？"
↓

 第一次 API 调用：client.chat_with_tools(messages, tools) 
 传入：对话历史 + 工具列表（含 get_current_time 的描述） 

↓

 模型返回：finish_reason = "tool_calls" 
 tool_calls = [{ "function": { "name": "get_current_time" } }]
↓

┌─────────────────────────────────────────────────────────┐
 代码执行：tool_registry.execute("get_current_time") 
 返回：{"time": "2026-04-22 10:50:33"} 
└─────────────────────────────────────────────────────────┘
↓
┌─────────────────────────────────────────────────────────┐
 追加 tool 消息到 messages： 
 { "role": "tool", "tool_call_id": "...", "content": ... }
└─────────────────────────────────────────────────────────┘
↓
┌─────────────────────────────────────────────────────────┐
  第二次 API 调用：client.chat(messages) （不带 tools） 
└─────────────────────────────────────────────────────────┘
↓
┌─────────────────────────────────────────────────────────┐
 模型生成最终回复： │
 "当前的具体时间是2026年4月22日10点50分33秒。" 
└─────────────────────────────────────────────────────────┘

## 技术收获

1. **理解 Function Calling 的本质**：模型只输出“调用意愿和参数”，实际执行由代码完成
2. **掌握 JSON Schema 在工具定义中的作用**：这是模型理解工具能力的“说明书”
3. **实现完整的工具调用闭环**：请求 → 解析 → 执行 → 回传 → 生成回复
4. **理解 Agent 能力边界**：Agent 的能力 = 模型自带知识 + 已注册的工具
